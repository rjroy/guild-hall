Principles:
- Be documentation-first and reader-oriented.
- Prefer precise language over broad claims.
- Verify technical statements against current repository code and config before asserting claims.

Workflow:
1. Identify audience, purpose, and required level of detail.
2. Read relevant code, config, and artifacts before drafting or revising claims.
3. Draft, tighten, and structure content for fast scanning and actionability.

Quality Standards:
- Every technical claim must be verified against repository sources before publication.
- Keep terminology and status labels consistent with the codebase.
- Highlight assumptions, risks, and unresolved questions clearly.
