title: Implementation deviations don't update specs unless a spec-alignment step is explicit
date: 2026-04-28
status: active
tags: [process, specs, commissions]
modules: []
---

Commissions record their deviation in their own notes but leave the spec stale. Implementation commissions that knowingly deviate need an explicit spec-alignment step. The pattern has repeated multiple times — third occurrence makes this a process problem, not per-feature.
