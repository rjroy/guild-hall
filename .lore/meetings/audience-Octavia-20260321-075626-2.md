**Meeting Notes: Guild Hall Infrastructure Spec Review with Octavia**

Session reviewed three infrastructure specifications to identify and resolve cross-reference inconsistencies. The event-router.md, commission-outcomes-to-memory.md, and context-type-registry.md specs contained inaccurate dependency claims and attribution errors. The event-router spec had incorrectly positioned outcome triage as a consumer (it subscribes directly to EventBus instead), while the other two specs attributed patterns to the event router that are generic DI conventions. Four edits across the three specs corrected these references, with commission-outcomes-to-memory.md and context-type-registry.md confirmed as implemented (only references changed), and event-router.md marked reopened due to a significant architectural revision in step 5 of the spec lifecycle.

Core concern raised: EventMatchRule in the current router spec is minimal—just `type` and `projectName`—raising the question of whether this is merely a thin wrapper around EventBus filtering. Discussion clarified that the router's architectural value lies not in matching sophistication but in enabling decoupled subscription patterns for multiple consumers. The revised spec positions the router as a generic `subscribe(rule, handler)` interface where the notification service becomes one consumer (registering handlers that dispatch to channels) and triggered commissions become another (without touching the router itself). This allows future matching enhancements without reimplementing matching for each consumer.

User endorsed the refactored approach ("Refactor then extend. It's my own rule.") and approved the event-router spec. Plan preparation for the code migration is beginning in a fresh session to avoid embedding unstated assumptions from the hot context.

**Key Decisions:** Approved refactored event-router spec with generic filtered subscription architecture and deferred matching complexity to exit points. Resolved loose threads in all three specs by removing false cross-dependencies.

**Artifacts:** event-router.md (spec revised, approved), commission-outcomes-to-memory.md (spec reference updates), context-type-registry.md (spec reference updates).

**Open Items:** Migration plan preparation starting; review of actual code refactoring steps pending plan completion.
