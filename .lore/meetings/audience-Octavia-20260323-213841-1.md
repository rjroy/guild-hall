MEETING NOTES: Guild Compendium Spec Scope Review
Date: 2026-03-24
Participants: User, Octavia (note-taking)

SUMMARY

Discussion centered on the scope framing of the Guild Compendium plugin package specification. The user identified that REQ-CMP-24 excluded the Illuminator worker based on narrow reasoning that "image generation work does not benefit from software craft knowledge," revealing a broader pattern: the spec was framing the compendium as "Guild Hall Software Craft Compendium" when the concept should be "Guild Hall Compendium" with domain-agnostic scope. The architecture already supports this (domain field in frontmatter is unrestricted), but examples, worker exclusions, and initial content list narrowed the framing unnecessarily. The user directed that the spec be widened to reflect the intended cross-domain nature of the compendium while keeping initial population focused on current gaps (software-related).

The widening required updates across seven locations in the spec. The core architectural change: expand worker opt-ins from four to six (adding Illuminator and Visionary), restrict exclusions to Researcher only (whose role is to produce knowledge, not consume guidance), and reframe package description and REQ language to reflect domain-agnostic craft knowledge rather than software-specific practices.

DECISIONS

Decision 1: Include Illuminator (Sienna) and Visionary (Celeste) in compendium opt-ins.
Reasoning: No technical barrier exists. The on-demand skill model means no context cost when entries in a domain do not exist. Visual entries (art styles, composition, image prompting) fit the same structure as software entries. Future expansion into any domain becomes possible without spec revision.

Decision 2: Restrict worker exclusions to Researcher only.
Reasoning: Researcher's exclusion remains grounded: her role is to produce fresh knowledge rather than apply standing craft knowledge. Illuminator and Visionary were excluded on assumptions about domain scope, which no longer apply.

Decision 3: Keep initial content research commissions software-focused but reframe as gap-driven, not boundary-enforcing.
Reasoning: Current commission output has documented gaps in spec writing, code review, TypeScript patterns, implementation, and prompt engineering. Initial entries address these gaps. Future entries can cover any domain without requiring spec change.
