**Guild Hall Meeting Notes: Commission Graph Refactoring**
**Audience: Octavia | Date: 2026-03-14**

Octavia reviewed the brainstorm document proposing replacement of the SVG-based CommissionGraph component with an HTML/CSS indented tree list. The current SVG uses a 370-line Sugiyama-style layout algorithm to render commission dependencies as a directed graph. The proposal simplifies this by using CSS connector lines for single-parent dependency chains while keeping multi-parent commissions flat with text annotations. The flat list already exists as the fallback when no inter-commission edges are present and reuses proven card patterns from DependencyMap.tsx.

Three open implementation questions were resolved. Sort order will keep it simple: root-level items sort by status priority via existing sortCommissions() function, with children appearing under their parents regardless of status. Dependency chains will not be collapsible, maintaining consistency with the existing non-collapsible card list and relying on the separate commission list filtering feature for status-based filtering. Performance concerns around parent lookup via adjacency list building are not considered a blocker; the O(n) operation is trivial and necessary.

**Decisions:**
Sort order within tree follows root-level status grouping with children under parents (Q1). No collapsibility for dependency chains (Q2). Parent lookup performance is acceptable (Q3).
