MEETING NOTES
Audience with Octavia
2026-03-21

Octavia reviewed the draft brainstorm on meeting view layout, which addresses two core problems: navigation friction (breadcrumb becomes unreachable after scrolling through long conversations) and vertical space pressure from a tall header. The brainstorm catalogs nine UI patterns with assessed trade-offs and proposes a three-phase implementation approach: Phase 1 switches the container to viewport-height (CSS-only), Phase 2 adds a condensable header, and Phase 3 refines mobile layout. The analysis identifies the key insight that the chat area's internal scroll already exists but isn't activated because the outer container lacks height constraints.

A critical issue was identified: the recommended approach hid agenda information on mobile devices rather than making it accessible in condensed form. This violates the principle that responsive design changes presentation, never data availability. Four corrections were made to the brainstorm: the condensed header state now truncates the agenda to a single line with an expand affordance instead of hiding it, the responsive strategy table was updated to explicitly document how users access full information at each breakpoint, Phase 3 language was clarified to emphasize relocation of sidebar content rather than removal, and Open Question #2 was marked as resolved by the accessibility constraint.

Key decisions made: all data present on desktop must remain reachable on mobile, changed only in presentation (collapsed, truncated, behind interaction). Phase 1 (viewport-height container) is identified as a low-risk, high-impact change that can ship quickly. Phase 2 (condensing header) requires component refactoring since MeetingHeader is currently a server component. Phase 3 (mobile refinements) depends on data about actual mobile usage patterns and remains conditional.

Artifacts: .lore/brainstorm/meeting-view-layout.md (revised). The brainstorm remains in draft status pending decision to move to specification or implementation.

Open items: frequency of mobile access to meetings, whether the 960px max-width constraint should be reconsidered for wider screens, and optimal placement of the "Close Audience" button in condensed and mobile layouts.
