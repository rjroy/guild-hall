**Meeting Notes: Triggered Commissions Specification Revision**

Octavia and the Guild Master discussed the architectural model for triggered commissions. Initial brainstorm assumed triggers would extend the global event router config, but Octavia clarified the preferred direction: triggered commissions should be commission artifacts (living in `.lore/commissions/` alongside scheduled commissions), project-scoped, and stateful. Each triggered commission tracks its own state: run count, last trigger timestamp, and last spawned commission ID. The artifact uses `type: triggered` with schedule-like state tracking fields (`runs_completed`, `last_triggered`, `last_spawned_id`) but excludes `repeat` to avoid complexity. Status values follow the commission lifecycle (active, paused, completed, failed), and approval mode (`auto` or `confirm`) determines how spawned commissions are handled.

The daemon evaluates triggered commissions by subscribing to the EventBus with a new trigger evaluator service (analogous to the scheduler service), matching events against registered triggers from scanned project artifacts, and spawning one-shot commissions via the commission creation API. This keeps triggers project-scoped and visible in the commissions list, consistent with Guild Hall's architecture.

A blocker emerged: the event router's current architecture (channels and notification rules in global `config.yaml`) conflicts with the broader Guild Hall design principle that project-specific configuration belongs in project artifacts. The event router implementation hardcodes this global model; before the triggered commissions spec can be finalized, the event router architecture must be revisited and resolved. Octavia will discuss this separately.

**Artifacts:** Review notes added to `.lore/specs/commissions/triggered-commissions.md` capturing the commission-artifact model decision, the blocker, and requirements triage for implementation upon event router resolution.

**Follow-up:** Event router architecture review and decision; then return to finalize triggered commissions spec.
