MEETING NOTES: Dashboard Selection Model Spec Review
Guild Hall | Octavia | 2026-03-15

SUMMARY

The team completed a detailed review and refinement of the Dashboard Selection Model specification. The spec defines a two-mode dashboard (All Projects and single-project focus) with consistent behavior across four cards: Briefing, In Flight, Recent Scrolls, and Pending Audiences. The review identified seven clarity gaps and consistency issues in the draft spec. All were addressed and resolved during the meeting, including clarifications on briefing cache logic, filter extraction scope, and architectural transitions from previous specs.

KEY DISCUSSIONS AND CORRECTIONS

The team confirmed that the spec implements Option B (LLM-synthesized briefing seeded from cached per-project briefings) directly, departing from the brainstorm's staged approach (Option C first, then B). This decision was documented in the spec overview for traceability. REQ-DASH-16 was corrected to specify sequential—not parallel—briefing generation for stale project caches, preventing multiple concurrent LLM calls. REQ-DASH-11 was pinned to "10 most recent items" instead of leaving N undefined, matching the existing `/workspace/artifact/document/list?limit=10` endpoint behavior. REQ-DASH-18 was clarified to explain the dual-condition cache validation logic: a briefing remains valid when either the git HEAD matches OR the entry is within TTL, meaning valid briefings can persist for weeks if no work has occurred. The briefing TTL is surfaced as a config value (`briefingCacheTtlMinutes`, default 60 minutes) rather than changing the cache logic itself. REQ-DASH-22 was updated to explicitly note the architectural shift of DependencyMap from server component to client component, providing context for implementers reading superseded specs.

ARTIFACTS

Brainstorm: Dashboard Selection Model (.lore/brainstorm/ui/dashboard-selection-model.md) – status resolved. Decisions summary updated to match spec implementation.

Spec: Dashboard Selection Model (.lore/specs/ui/dashboard-selection-model.md) – status draft. 23 requirements across selection model, In Flight card, Recent Scrolls, Pending Audiences, briefing synthesis, and superseded requirements. All success criteria and validation tests defined.
