MEETING NOTES: Artifact Smart Views Specification Review
Audience: Guild Hall with Octavia
Date: 2026-03-21

DISCUSSION SUMMARY

A comprehensive documentation review was conducted on the Artifact Smart Views specification (`.lore/specs/ui/artifact-smart-views.md`). The review identified seven substantive findings across clarity, completeness, and consistency categories. The spec defines three navigation filters for the artifacts tab ("What's Next?", "Needs Discussion", "Ready to Advance") intended to replace directory-tree-first browsing with intent-based views that cut across artifact types. A critical contradiction was discovered: the spec claimed "What's Next" was a superset of the other two views, but the filter definitions (showing only Group 0 artifacts) contradicted this, as "Needs Discussion" included active research artifacts (Group 1). Additionally, the gem mapping correction for `approved` (moving it from green/Group 1 to orange/Group 0) conflicted with the artifact-sorting spec, which owns the ARTIFACT_STATUS_GROUP data structure.

The review also flagged missing directory mappings for `meetings/` and `commissions/` directories in the type-to-label translation, unacknowledged exclusion of several artifact types (retros, diagrams, notes, references) from all smart views, and unclear language around which brainstorm statuses should be included in "Needs Discussion" filters. The AI validation section was found to be underspecified for edge cases like `parked` brainstorms and `wontfix` issues.

DECISIONS AND REASONING

Decision: "What's Next" filter will include both Group 0 (pending/orange, "waiting on human action") and Group 2 (blocked/red, "something needs attention"). Reasoning: Both groups represent artifacts that require human attention and action, making them equally valid for an "actionable items" view. This resolves the false superset claim and provides users with a more complete view of work needing their attention.
