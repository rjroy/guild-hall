**MEETING NOTES: Heartbeat Commission Dispatch Spec Review**

The session focused on refining the Heartbeat Commission Dispatch specification following a draft review. Two substantive changes were made to the File Scaffolding and GM session behavior sections. REQ-HBT-25 was updated to ensure the daemon actively maintains the heartbeat file's instructional header by replacing everything before the first `##` with the template on every initialization, preventing drift from worker modifications. Separately, REQ-HBT-9 was expanded with a cleanup instruction directing the GM to commission heartbeat maintenance work when the file grows unwieldy rather than attempting inline fixes. A parallel spec review by fresh-context agent uncovered five issues: a scaffolding test that contradicted the header maintenance requirement, two incorrect cross-reference pointers, missing rationale for the `initiate_meeting` tool's internal routing, and an unhandled edge case when no standing orders exist. All five were fixed: the scaffolding test now validates header repair with section preservation, REQ-HBT-34 now correctly points to REQ-HBT-42 for type simplification details, REQ-HBT-21 now correctly points to REQ-HBT-39 for orchestrator removals, REQ-HBT-10 gained a note on `initiate_meeting` dependency injection, and REQ-HBT-9 added explicit handling for the empty-orders-with-activity case.

The spec is now internally consistent with correct cross-references, realistic test expectations, and clear guidance for both daemon behavior and GM decision-making under edge cases. No architectural decisions were revisited; all changes were clarity and correctness focused.
