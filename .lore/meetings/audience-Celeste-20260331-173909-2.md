MEETING NOTES: Heartbeat Commission Dispatch - Open Questions Resolution

SUMMARY

Celeste and team reviewed all five open questions from the heartbeat commission dispatch brainstorm. The discussion confirmed the architectural direction: replace scheduled commissions and triggered commissions with a simpler heartbeat system that uses natural language standing instructions in a `.lore/heartbeat.md` file, evaluated by a Guild Master session on Haiku.

The session reframed the heartbeat dispatcher as a Guild Master capability (not a standalone Haiku system), which simplified the architecture significantly. The heartbeat file will be stored per-project, version-controlled, edited through the existing Artifacts tab, with no new UI forms needed. All remaining questions converged on simple, practical solutions.

DECISIONS

1. Heartbeat file location: Per-project `.lore/heartbeat.md`. Rationale: Standing instructions are project-specific, version-controlled with the project, and editable by both users and agents. No global file needed.

2. Haiku session design: Guild Master on Haiku, not a separate system. The heartbeat reuses the same GM toolbox as briefing generation—commission creation is already there. No new tool surface required.

3. User interface: Artifacts tab handles all heartbeat editing. Schedule and trigger creation forms are removed entirely. The heartbeat file self-documents its purpose with instructions at the top.

4. Tick interval: Configurable in `~/.guild-hall/config.yaml` with 1-hour default. Dashboard includes per-project `[Tick Now]` button (alongside existing `[View]` buttons) showing heartbeat file size. Manual ticks support testing without waiting for the next interval.

5. Cost and rate limiting: No retry on rate limit; fail gracefully. Keep prompt discipline for quality, not cost. Show heartbeat file size as a visibility signal. Optimization: skip Haiku call entirely if heartbeat has no content after instructional headers.

ARTIFACTS PRODUCED

.lore/brainstorm/heartbeat-commission-dispatch.md — Comprehensive brainstorm covering six proposals, rationale, vision alignment, tradeoffs, and all five open questions with resolutions. Ready for spec handoff.

FOLLOW-UP

Spec commission should cover: removal scope for scheduler and trigger evaluator; heartbeat file format and scaffolding; daemon loop implementation; event condensation format; trust markers (auto/confirm); provenance tracking; and graceful degradation on rate limits.
