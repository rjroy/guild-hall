# Task: Add On-Demand Manager Briefing with SDK Generation and Caching

## What

On-demand natural language briefing generated by a single-turn SDK session, cached for 1 hour per project.

**daemon/services/briefing-generator.ts:**

`createBriefingGenerator(deps)` returns `{ generateBriefing(projectName): Promise<string>, invalidateCache(projectName): void }`.

`generateBriefing()` flow:
1. Check in-memory cache (Map keyed by projectName, 1-hour TTL).
2. On miss: call `buildManagerContext()` (Task 4's module) to assemble current state.
3. Run a single-turn SDK session with a briefing prompt: "Based on the current state of this project, provide a concise briefing (3-5 sentences) covering: what's in progress, what's blocked, what recently completed, and what needs attention next."
4. Cache and return the text response.

Fallback when no `queryFn` (SDK not installed): return a template-based summary assembled from the context data (commission counts by status, recent completions). This satisfies REQ-VIEW-12.2's "when no manager is configured" case.

Follow the notes-generator pattern from Phase 3: discriminated union return type, `maxTurns: 1`, `maxBudgetUsd: 0.10`.

**daemon/routes/briefing.ts:**

`GET /briefing/:projectName` returns `{ briefing: string, generatedAt: string, cached: boolean }`.

**app/api/briefing/[projectName]/route.ts:**

Next.js API route proxying to the daemon endpoint.

**components/dashboard/ManagerBriefing.tsx update:**

Replace the stub ("No Guild Master configured.") with a client component that:
1. Fetches from `/api/briefing/<projectName>` on mount.
2. Shows a loading skeleton while fetching.
3. Renders briefing text in the existing Panel with scroll-window divider.
4. Shows "Last updated: <time>" from `generatedAt`.
5. For initial implementation, show the first project's briefing. Multi-project briefing aggregation is future work.

**Production wiring (daemon/app.ts):**

Create `briefingGenerator` with `createBriefingGenerator()`, mount `createBriefingRoutes({ briefingGenerator })`.

## Validation

- SDK-generated briefing uses correct prompt and returns text
- Cache hit returns cached text without triggering SDK call
- Cache expires after 1 hour, triggering new generation
- `invalidateCache()` clears the entry for the given project
- Fallback when no queryFn: produces template summary from context data (not an error)
- Route returns briefing text with `generatedAt` timestamp and `cached` flag
- Empty project (no commissions/meetings) returns a minimal briefing (not an error)
- ManagerBriefing component fetches on mount and renders the briefing text
- ManagerBriefing shows loading state while fetching
- Production wiring creates briefing generator and mounts routes
- All tests pass, linting clean, commit created (triggers pre-commit hook validation)
